\section secParallelRenderingWithParaView Parallel rendering with ParaView
<hr>

<b>Prerequisite</b>: <a href="http://paraview.org/">ParaView</a> (client) is installed on your local machine,
and ParaView (server; with MPI support) is installed on the machine which is intended as rendering server.

<b>Important</b>: The version of the client must match that of the server!

For a local installation of ParaView on the rendering server see \ref secInstallParaView.

This mode of running ParaView is also called "Client-Server mode".
In client-server mode, one executes the <tt>pvserver</tt> program on a parallel machine and connect to it with the ParaView client application.

<ol>
<li> Start the ParaView server on the rendering server, e.g. cekon, with your local machine as client (the parameter <tt>-ch</tt> describes the "client host"):
     \verbatim
     salloc -n <NP> mpirun /usr/local/bin/pvserver -ch=<yourlocalhost>.gcsc.uni-frankfurt.de -rc --use-offscreen-rendering
     \endverbatim
     The option <tt>-rc</tt> stands for "reverse connection". This means that the server connects to the client instead of the client connecting to the server.
     This may be necessary when the server is behind a firewall.
     </li>

<li> Start the ParaView client on your local machine as usual.
     </li>

<li> Then choose "File -- Connect".
     In the new dialog "Choose Server Configuration" click "Add Server" (window title changes to "Edit Server Configuration").

     Fill in the necessary informations to define a connection to the ParaView server running on cekon:
     <ul>
     <li> Name: "cekon" (only descriptive; this is the name that will appear in the "Choose Server Configuration" dialog),
     <li> Server Type: "Client / Server (reverse Connection)",
     <li> Port: "11111" (if default port can be used; see below).
     </ul>
     Then click "Configure" and choose "Startup Type": "Manual". Click "Save"

     Select the (newly defined) server and click "Connect".
     </li>

<li> In the "Pipeline Browser" of your client choose the server as "data object" -- TODO: Test and explicate!
     </li>


</ol>
If a ParaView server is already running and listening at the default port, <tt>11111</tt>
(or this port is blocked by a firewall etc.), you have to start the server using a non-default port
using the additional parameter <tt>--server-port=&lt;portnumber&gt;</tt> (and of course take this into account also when connecting from the client to the server).


For some further documentation see e.g.:
     <ul>
     <li> http://www.paraview.org/Wiki/Setting_up_a_ParaView_Server#Running_the_Server,</li>
     <li> http://www.vtk.org/Wiki/Setting_up_a_ParaView_Server,</li>
     <li> http://www.fz-juelich.de/ias/jsc/EN/Expertise/Support/Visualization/ScientificVisualization/JUVIS/_node.html,</li>
     <li> http://www.fz-juelich.de/ias/jsc/EN/Expertise/Support/Visualization/ScientificVisualization/JUVIS/SampleSessionLocalRender.html.</li>
     </ul>
